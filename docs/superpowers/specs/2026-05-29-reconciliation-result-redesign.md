# 核对结果页重设计 — 设计文档

## 背景

当前核对结果页存在以下问题：
1. "风险与未匹配"是垃圾桶，混杂工时不一致、Excel有PDF无、疑似姓名匹配等不同状态
2. 仓库核对不能下钻差异归因（用户不知道"差$41"来自哪些人）
3. 仓库级（差$41）和员工级（差$85K）给出矛盾信号，系统没有解释
4. 没有结论层，用户需要自己判断"到底有没有问题"
5. Excel差异报告结构与前端不一致

## 目标

用户打开结果页，**一眼看到结论，一眼知道差异点**。

## 设计

### 1. 结论区（页面顶部）

```
✅ 仓库总金额核对通过    差异: -$41.30 (0.03%)
⚠️ 2人工时/金额差异需关注
📋 本批发票覆盖 34人，账单共 125人（91人不在本批发票）
```

显示规则：
- 总金额差异 < 0.1% → 绿灯"通过"
- 有金额差异员工 → 黄灯"需关注"
- 有低置信度抽取 → 红灯"需人工复核"
- "不在本批发票"只显示数字，不标红

### 2. 仓库概览 + 差异归因下钻

仓库表格，每行显示仓库号、PDF总金额、Excel总金额。

差异 < $1 的仓库：归因行显示"无显著差异员工"。
差异 >= $1 的仓库：自动展开归因，列出贡献差异最大的员工（按金额绝对值排序），末尾"其他N人"聚合剩余差异。

仓库行可点击折叠/展开。

数据来源：`warehouseComparison.rows[].employeeRows` 中筛选 `matchStatus === "金额差异"` 的行，按 `amountDelta` 绝对值降序排列。

### 3. 待处理事项（分组展示，按优先级排序）

三组独立折叠区域：

**组1：工时不一致**（默认展开）
- 筛选 `matchStatus === "工时不一致"` 的行
- 显示员工名、PDF工时、Excel工时、差值

**组2：姓名格式差异**（默认展开）
- 使用 `candidateMatches` 数据
- 显示 PDF姓名、Excel姓名、相似度、PDF金额、Excel金额
- 不需要确认按钮，纯展示

**组3：不在本批发票**（默认收起）
- 筛选 `matchStatus === "Excel有PDF无"` 的行
- 只显示人数和总金额，点击展开查看完整列表

### 4. 删除的展示

- 删除"金额差异员工"独立表（合并到仓库归因下钻中）
- 删除"低置信度抽取"独立表（合并到结论区的红灯提示中）

### 5. Excel 差异报告改进

新增"核对结论"sheet作为第一个sheet：
- 核对结论（通过/需关注/需人工复核）
- 总金额差异和百分比
- PDF覆盖人数 vs 账单总人数
- 各仓库差异归因摘要

现有sheet改名：
- "未匹配员工" → "不在本批发票"
- "未匹配候选" → "姓名格式差异"
- 其余保持不变（向后兼容）

### 6. 后端 API 变更

comparisonSummary 新增字段：
- `conclusionLevel`: "pass" | "warning" | "critical"
- `conclusionMessage`: 一句话结论
- `pdfEmployeeCount`: PDF抽取人数
- `excelEmployeeCount`: 账单总人数

warehouseComparison.rows 新增字段：
- `attribution`: 差异归因数组，每项含 `{employeeName, pdfAmount, excelAmount, delta}`

## 不在范围内

- 姓名匹配的在线确认/拒绝功能（不需要）
- 修改 compare.py 的核对逻辑本身（只改展示层）
- 修改抽取逻辑

## 验收标准

1. 打开结果页，3秒内看到结论（通过/需关注/需人工复核）
2. 仓库差异 >= $1 时，自动展开显示差异归因到具体员工
3. "不在本批发票"默认收起，不干扰主要核对流程
4. Excel报告第一个sheet是"核对结论"，包含归因摘要
5. 现有批次数据重新渲染后，展示效果与设计一致
