# 海外劳务工报账核对工作台设计规格

## 目标

在西格玛工作台首页新增独立入口“海外劳务工报账核对”，用于薪酬组小范围核对海外劳务供应商发票与线下账单。第一版聚焦一个供应商、一个账期、多张 PDF 发票和一份线下账单 Excel，自动提取员工工时与金额，按员工汇总比对，并输出金额差异和风险明细。

该功能是新的业务场景，不复用招聘奖金或内推奖金核算规则。现有西格玛工作台只作为发布入口和应用承载壳。

## 范围

第一版包含：

- 首页新增“海外劳务工报账核对”工作台入口。
- 独立劳务核对批次，不与招聘奖金核算批次混用。
- 每个批次处理一个供应商、一个账期、多张 PDF 发票和一份已按供应商筛选好的线下账单 Excel。
- 用户手动选择 Excel 工作表。
- 平台自动推荐 Excel 字段映射，用户确认姓名、工时、金额列。
- 通过 AI API 抽取 PDF 发票员工明细。
- 按员工汇总 PDF 与 Excel 两侧的工时和金额。
- 用确定性代码进行姓名匹配、工时辅助校验、金额差异判断。
- 金额差异绝对值大于 0.01 才列为异常。
- 低置信度 AI 抽取项不阻断比对，但在页面和导出报告中标记风险。
- 导出 Excel 差异报告，供薪酬组复核和留档。

第一版不包含：

- 本地大模型或私有化 AI 部署。
- 多供应商混合批次自动拆分。
- 供应商主数据、合同费率库或自动付款审批。
- AI 自动决定最终对错或直接改写金额。
- 与招聘奖金核算结果合并。

## 产品流程

1. 用户从西格玛工作台首页点击“海外劳务工报账核对”。
2. 用户进入独立劳务核对工作台，创建新批次。
3. 用户填写供应商、账期、币种和可选备注。
4. 用户上传该供应商的一批 PDF 发票。
5. 用户上传一份已按供应商筛选好的线下账单 Excel。
6. 用户选择 Excel 工作表。
7. 平台读取表头和样例行，推荐姓名列、工时列、金额列和可选币种列。
8. 用户确认或修改字段映射。
9. 平台从 PDF 提取文本，调用 AI API 抽取员工明细。
10. 平台按员工汇总 PDF 与 Excel 两侧数据。
11. 平台执行确定性比对，生成金额差异、工时风险、未匹配、低置信度风险和通过项。
12. 用户查看结果页，必要时检查低置信度和未匹配项。
13. 用户导出差异报告 Excel。

## 模块边界

新增独立劳务核对模块，不放入现有招聘奖金计算引擎。

- `LaborWorkbench`：前端入口、批次列表、上传、字段映射、结果复核和导出入口。
- `LaborRun`：劳务核对批次，保存供应商、账期、上传文件、字段映射、AI 抽取结果、比对结果和批次状态。
- `InvoiceExtractor`：PDF 发票抽取器，负责文本提取、AI API 调用、结构化 JSON 校验和置信度记录。
- `WorkbookMapper`：Excel 线下账单映射器，负责工作表读取、字段推荐、字段映射保存和明细抽取。
- `LaborMatcher`：员工匹配器，负责姓名标准化、工号优先匹配、姓名匹配、工时辅助匹配和疑似匹配识别。
- `LaborComparator`：确定性比对器，负责员工维度汇总、工时差异、金额差异和结果分类。
- `LaborReportExporter`：报告导出器，负责生成可复核的 Excel 差异报告。

AI 只存在于 `InvoiceExtractor`，不进入匹配和比对决策。最终差异由代码按规则计算。

## 匹配规则

第一版按员工汇总，不按仓库、发票号或日期拆分。

匹配顺序：

1. 如果 PDF 和 Excel 两边都有工号，优先用工号匹配。
2. 如果缺少工号，用标准化姓名精确匹配。
3. 如果姓名不完全一致，但姓名相似且工时差异绝对值小于或等于 0.1 小时，标记为“疑似姓名匹配”。
4. 如果仍无法匹配，分别输出“PDF 有 Excel 无”和“Excel 有 PDF 无”。

姓名标准化规则：

- 大小写统一。
- 去除前缀编号，例如 `#1`。
- 去除括号备注，例如 `(Fairway)`。
- 支持 `LAST, FIRST` 转换为 `FIRST LAST`。
- 去除多余空格、常见标点和重音符。
- 保留原始姓名用于追溯。

## 比对规则

- 工时用于辅助匹配和风险判断。
- 金额差异是主要异常条件。
- 金额差异绝对值小于或等于 0.01 视为一致。
- 金额差异绝对值大于 0.01 输出到“金额差异员工清单”。
- 工时差异绝对值小于或等于 0.1 小时视为辅助匹配通过。
- 工时不一致但金额一致，输出为“工时风险项”。
- 低置信度 PDF 行不阻断比对，但结果和报告必须标记风险。

结果分类：

```text
金额差异
工时不一致
疑似姓名匹配
PDF有Excel无
Excel有PDF无
低置信度抽取
通过
```

## AI 接入

第一版通过云端 AI API 处理发票抽取，暂不考虑本地或私有化模型。

基础配置：

```text
AI_ENABLED=true
AI_PROVIDER=openai_or_azure
AI_API_KEY=<required-secret>
AI_BASE_URL=<required-endpoint>
AI_MODEL=<required-model>
AI_TIMEOUT_SECONDS=90
AI_MAX_PAGES_PER_REQUEST=5
AI_CONFIDENCE_THRESHOLD=0.85
AI_AMOUNT_TOLERANCE=0.01
LABOR_HOURS_TOLERANCE=0.1
```

调用策略：

- 优先从 PDF 抽取可复制文本，把文本发送给 AI。
- 如果 PDF 无文本或文本质量差，后续版本可走 OCR 或图片抽取兜底。
- 按文件、页或文本块分批请求，避免单次请求过大。
- AI 必须返回结构化 JSON，不接受自由文本作为正式结果。
- 每条员工明细必须包含置信度和原文依据。
- AI 失败时，批次状态进入“抽取失败”，用户可重试或下载错误信息。

## 前端体验

后续实现前端时使用 `frontend-design` 原则，视觉方向为“审计级工作台 + 清晰风险分层 + 文件证据感”。

首页新增独立卡片：

- 标题：海外劳务工报账核对
- 副标题：AI 抽取供应商发票，按员工核对工时与金额差异
- 状态提示：支持 PDF 发票、Excel 账单、差异报告导出

劳务核对页面采用步骤式工作台：

1. 新建批次
2. 上传文件
3. 字段映射
4. 抽取与比对
5. 结果复核

结果页包含：

- 总览指标：PDF 总人数、Excel 总人数、PDF 总工时、Excel 总工时、PDF 总金额、Excel 总金额、金额差异总额、异常人数。
- 主差异表：金额差异大于 0.01 的员工。
- 风险表：工时不一致、疑似匹配、未匹配、低置信度抽取。
- 明细抽屉：展示 PDF 文件名、页码、原文片段和 Excel 行号。

视觉上应保持专业、克制、可审计。AI 能力隐藏在“发票抽取”步骤里，不做聊天式界面，不用泛化的后台模板样式。

## 导出报告

导出 Excel 报告包含以下工作表：

```text
核对摘要
金额差异员工
工时风险项
未匹配员工
低置信度抽取
PDF抽取明细
Excel账单明细
字段映射记录
```

报告必须保留来源引用，便于薪酬组复核：

- PDF 文件名
- PDF 页码
- PDF 原文片段
- Excel 工作表名
- Excel 行号
- 字段映射记录
- AI 置信度

## 异常处理

- PDF 无法读取：标记具体文件失败，不影响其他文件解析。
- AI 返回 JSON 不合法：重试一次，仍失败则进入抽取失败。
- Excel 工作表为空：阻断字段映射。
- 字段映射缺少姓名、工时或金额：阻断比对。
- 金额或工时无法转数字：进入风险项，不静默丢弃。
- 员工姓名重复：按员工汇总，但保留来源明细。
- 低置信度：不阻断，结果和报告标记风险。

## 测试策略

- PDF 抽取样本测试：用 US Elogistics 样本验证多张 PDF 员工行抽取。
- Excel 映射测试：覆盖不同列名、不同列顺序、空列和金额格式。
- 姓名标准化测试：覆盖 `LAST, FIRST`、`#1`、括号备注、尾号、大小写和重音符。
- 比对测试：覆盖金额容差 0.01、超过容差、工时不一致、未匹配、重复员工。
- API 测试：覆盖批次创建、文件上传、字段映射、AI 抽取、比对和导出。
- 前端流程测试：覆盖首页入口、步骤流、字段确认、结果筛选、明细抽屉和报告导出。

## 验收标准

- 首页有独立“海外劳务工报账核对”入口。
- 用户可以完成一个供应商批次的 PDF 和 Excel 上传。
- 用户可以选择 Excel 工作表并确认姓名、工时、金额字段。
- PDF 发票员工明细通过 AI API 抽取并保留置信度和原文依据。
- 平台能按员工汇总 PDF 与 Excel 两侧的工时和金额。
- 金额差异大于 0.01 的员工会进入异常清单。
- 低置信度项不阻断结果生成，但在页面和报告中可见。
- 用户可以导出包含摘要、差异、风险、明细和字段映射记录的 Excel 报告。
